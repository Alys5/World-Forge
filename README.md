# World-Forge: The Orchestrator Pipeline

*Universal Agentic Roleplay World Building & SillyTavern Export*

World-Forge is an advanced, multi-agent pipeline designed to automate the creation of rich, complex, and modular roleplay worlds for [SillyTavern](https://github.com/SillyTavern/SillyTavern). By leveraging a team of specialized AI personas (agents), the pipeline takes you from a raw idea to a fully functioning, highly detailed world package ready for immediate import.

## What It Produces

The pipeline automatically generates a complete SillyTavern package, including:
- **Character Cards** (`.json`) with behavioral instructions and depth prompts.
- **Tier 1 Lorebooks:** Permanent world rules, factions, and locations.
- **Tier 2 Lorebooks:** Protagonist identity, character backgrounds, and intimacy profiles (when applicable).
- **Tier 3 Lorebooks:** Modular, arc-specific state management and intimacy registers.
- **Group Lorebooks:** A combined lorebook for easy management in SillyTavern.
- **Chat Completion Presets:** Optimized runtime settings for your specific world.

## Intended Usage

World-Forge is designed to be used as a workspace in **Visual Studio Code (VS Code)** using an AI assistant extension like **Roo Code**.

### How to Get Started

1. **Clone the Repository:** Open this repository as a workspace in VS Code.
2. **Launch Roo Code:** Open your Roo Code (or equivalent) agent chat.
3. **Start the Forge:** Type the following command in the chat to begin:
   ```text
   /worldforge start
   ```
4. **Follow the Prompts:** The first agent (`The Interviewer`) will interactively walk you through the World Seed creation. From there, the agents will pass the torch to one another until the world is fully compiled.

### File Explanations

**Root Files:**
- `World_Seed.md`: The foundational document produced by Phase 0. Contains all the core ideas, characters, rules, and arcs for your world.
- `UNRESOLVED_QUESTIONS.md`: Generated by Phase 1 if there are logical gaps or missing details in your World Seed that require your input before drafting can begin.
- `UNRESOLVED_INTIMACY.md`: Generated by Phase 2.5 if intimacy scenes are intended but missing specific details.
- `Notes_On_functionality.md`: Contains authoritative documentation on SillyTavern's inner functionality and runtime mechanics. Note: This dictates how SillyTavern works, not how the World-Forge pipeline operates.

**Drafts/ (Work-in-Progress Content):**
- `Master_Design.md`: The structured blueprint created by Phase 1, organizing your World Seed into concrete Tiers.
- `Card_[CharName].md`: The draft character card content for a specific character.
- `User.md`: The draft `{{user}}` Persona Description text — the always-on identity floor the human pastes into SillyTavern's Persona Description field. SillyTavern provides no import format for personas, so the pipeline produces a paste-ready text block instead. Paired with the Tier 2 Protagonist Lorebook. Excludes voice/personality/manner — those are the human's domain.
- `Tier1_World_Entries.md`: Drafts of permanent, fundamental world lore and rules.
- `Tier2_[ProtagonistName]_Entries.md` & `Tier2_[CharName]_Entries.md`: Drafts of character backgrounds, identity references, and permanent traits.
- `Tier2_[CharName]_Intimacy_Profile.md`: Drafts of a character's permanent intimate preferences, boundaries, and physical reactions.
- `Tier3_Arc[N]_[Title]_Entries.md`: Drafts of state changes, rules, and world states for a specific story arc.
- `Tier3_Arc[N]_Intimacy_Register.md`: Drafts of how intimacy functions thematically and mechanically within a specific arc.
- `Instructions_[CardName].md`: The drafted system prompts, depth prompts, and behavioral instructions for the LLM.
- `Editor_Critique_[Round N].md`: Feedback generated during Phase 3's structural validation loops.
- `Voice_Audit_Report`, `Arc_Transition_Audit`, `Intimacy_Audit_Report`: Feedback and pass/fail reports from the Phase 3 validation auditors.

**workflows/ (Pipeline Instructions):**
- `world-forge.md`: The main pipeline instructions document. This is the master file that your AI agent references to run the agentic start and guide the entire process.

**templates/ (Reference Materials):**
- Contains `.md` and `.json` reference files used by The Compiler and The Prompt Engineer to ensure all output is correctly formatted for SillyTavern.

**Export/ (Final Deliverables):**
- `[CharName]_Card.json`: The final Character Card ready to import into SillyTavern.
- `User.md`: The `{{user}}` Persona Description text, passed through unchanged from `Drafts/User.md`. After importing the rest, open ST → User Settings → Persona Management, create your persona, paste the block between `--- BEGIN PERSONA DESCRIPTION ---` and `--- END PERSONA DESCRIPTION ---` into the persona's Description field, and link `[ProtagonistName]_Lorebook.json` in the persona's Lorebook field.
- `World_Lorebook.json`, `[CharName]_Lorebook.json`, `[ProtagonistName]_Lorebook.json`: Permanent lorebooks representing Tier 1 and Tier 2 content.
- `Arc[N]_Lorebook.json`: Modular lorebooks to swap in and out as the roleplay progresses through different arcs.
- `[CharName]_Intimacy_Profile.json` & `Arc[N]_Intimacy_Register.json`: The compiled JSON lorebooks managing intimacy mechanics.
- `Group_Lorebook.json`: A single, comprehensive file containing all the tiered lorebooks combined for easy group management in SillyTavern.
- `Compiler_Log.md` & `Prompt_Engineer_Audit.md`: Final audit logs ensuring all tokens, positions, and injection blocks are correct.
- `[WorldName]_ChatPreset.json`: The tailored Chat Completion Preset, optimizing the LLM's runtime settings and generation instructions for your specific world.

## The Pipeline Phases

The World-Forge process is broken down into structured phases. Each phase is handled by a dedicated agent with a specific role:

- **Phase 0: The Interviewer** – Walks you through discovery, asking probing questions to define the tone, characters, arcs, and world rules.
- **Phase 1: The Refiner** – Structures your ideas into Tiers and flags any gaps before drafting begins.
- **Phase 2: The Architect** – Drafts the raw content (Character Cards, Lorebooks, System Prompts).
- **Phase 2.5: The Intimacy Architect** *(Optional)* – Drafts character intimacy profiles and arc registers, if your world includes intimate scenes.
- **Phase 3: The Editor** – Iteratively validates prose quality, tier integrity, and LLM instructions.
- **Phase 3.5, 3.6, 3.7: The Auditors** – Parallel validation phases that test behavioral fidelity, continuity across story arcs, and intimacy realism.
- **Phase 4: The Compiler** – Translates the approved Markdown drafts into SillyTavern-ready JSON files.
- **Phase 5: The Prompt Engineer** – Audits the final output for runtime correctness and generates a tailored Chat Completion Preset.

## Human Pause Gates

The pipeline is entirely agentic, but it knows when to stop and ask for your input. It will automatically pause if it finds missing information, logical conflicts, or if you need time to flesh out a concept. You can resume the pipeline using the trigger commands.

## Trigger Commands

Use these commands in your Roo Code chat to control the pipeline:

| Command | Action |
|---|---|
| `/worldforge start` | Begin from Phase 0 (The Interviewer) |
| `/worldforge resume phase0` | Resume the interview from the last completed section |
| `/worldforge resume phase1` | Re-run The Refiner with answered questions |
| `/worldforge resume phase2` | Re-run The Architect |
| `/worldforge resume phase2.5` | Re-run The Intimacy Architect |
| `/worldforge resume phase3` | Re-enter The Editor loop |
| `/worldforge resume phase[3.5/3.6/3.7]` | Re-run the respective Auditor |
| `/worldforge resume phase4` | Re-run The Compiler |
| `/worldforge resume phase5` | Re-run The Prompt Engineer |
| `/worldforge status` | Report the current phase, round, and open blockers |
| `/worldforge skip phase0` | Begin from Phase 1 (if you wrote the World Seed manually) |
| `/worldforge skip phase[2.5/3.7]` | Skip intimacy phases if your world has no intimate content |
